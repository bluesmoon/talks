When should you sign the CLA?
----------------
[bg artolog/2519646167_1b96c5b87b.jpg]
- Any intellectual property requires a signed CLA, eg:
   - [class="light"] Code, including any significant patch or feature addition
   - [class="light"] Documentation, including any writing 500 words or longer
   - [class="light"] Examples/demo code that becomes part of the YUI distribution
   - [class="light"] Graphics or other visual creative work that becomes part of the YUI distribution
%

1a. Code Patterns: Private variables visible
----------------
[bg krisgriffon/http://farm1.static.flickr.com/21/26220260_7ae6f36f4c_b_d.jpg]
<pre class="dark">
   YAHOO.util.Foo = <span style="color:#ffdead;">function</span>() <span style="color:#ffdead;">{</span>
        <span style="color:#fa8072;">this</span>._foo = 10;
        <span style="color:#fa8072;">this</span>._bar = <span style="color:#87ceeb;">&quot;Hello, World&quot;</span>;

        <span style="color:#6495ed;font-weight:bolder;">return</span> <span style="color:#fa8072;">this</span>;
   <span style="color:#ffdead;">}</span>;

   <span style="color:#fa8072">var</span> foo = <span style="color:#ffdead;font-weight:bolder;">new</span> YAHOO.util.Foo();

   foo._foo = 20;
</pre>
%
1b. Code Patterns: Private variables hidden
----------------
[bg krisgriffon/http://farm1.static.flickr.com/21/26220260_7ae6f36f4c_b_d.jpg]
<pre class="dark">
   (<span style="color:#ffdead">function</span>() <span style="color:#ffdead">{</span>
        <span style="color:#fa8072">var</span> U=YAHOO.util, E=YAHOO.util.Event;
        <span style="color:#fa8072">var</span> l_foo, l_bar;

        U.Foo = <span style="color:#ffdead">function</span>() <span style="color:#ffdead">{</span>
                l_foo = 10;
                l_bar = <span style="color:#87ceeb">&quot;Hello, World&quot;</span>;

                <span style="color:#6495ed;font-weight:bolder;">return</span> <span style="color:#fa8072">this</span>;
        <span style="color:#ffdead">}</span>;
   <span style="color:#ffdead">}</span>());

   <span style="color:#fa8072">var</span> foo = <span style="color:#ffdead;font-weight:bolder;">new</span> YAHOO.util.Foo();

   <span style="color:#888;">// No access to l_foo</span>
</pre>
%
2a. Code Patterns: Private methods exposed
------------------------------------------
[bg sovietuk/http://farm1.static.flickr.com/61/154340012_0445caa0bd_b_d.jpg]
<pre class="dark">
   YAHOO.util.Foo = <span style="color:#ffdead;">function</span>() <span style="color:#ffdead;">{</span>
        <span style="color:#fa8072;">this</span>._init();

        <span style="color:#6495ed;font-weight:bolder;">return</span> <span style="color:#fa8072;">this</span>;
   <span style="color:#ffdead;">}</span>;

   YAHOO.util.Foo.prototype = <span style="color:#ffdead;">{</span>
        _init: <span style="color:#ffdead;">function</span>() <span style="color:#ffdead;">{</span>
             <span style="color:#888;">// private method</span>
        <span style="color:#ffdead;">}</span>
   <span style="color:#ffdead;">}</span>;
</pre>
%
2b. Code Patterns: Private methods hidden
-----------------------------------------
[bg sovietuk/http://farm1.static.flickr.com/61/154340012_0445caa0bd_b_d.jpg]
<pre class="dark">
   (<span style="color:#ffdead;">function</span>() <span style="color:#ffdead;">{</span>
        <span style="color:#fa8072;">var</span> _init = <span style="color:#ffdead;">function</span>() <span style="color:#ffdead;">{</span>
                <span style="color:#888;">// private method</span>
        <span style="color:#ffdead;">}</span>;

        YAHOO.util.Foo = <span style="color:#ffdead;">function</span>() <span style="color:#ffdead;">{</span>
                _init.call(<span style="color:#fa8072;">this</span>);

                <span style="color:#6495ed;font-weight:bolder;">return</span> <span style="color:#fa8072;">this</span>;
        <span style="color:#ffdead;">}</span>;
   <span style="color:#ffdead;">}</span>());
</pre>

3. Code Patterns: Extending v/s Augmenting
-------------------------------------------
[bg thevince/http://farm4.static.flickr.com/3031/2557418141_8145b77c15_b_d.jpg]
<pre class="dark small">
   YAHOO.lang.extend(derived, base, overrides);

   YAHOO.lang.augment(object, mixin, overrides);
</pre>
%
3. Code Patterns: Extending v/s Augmenting
-------------------------------------------
[bg thevince/http://farm4.static.flickr.com/3031/2557418141_8145b77c15_b_d.jpg]
<pre class="dark small">
   YAHOO.lang.extend(derived, base, overrides);

   YAHOO.lang.augment(object, mixin, overrides);
</pre>
- Use extend for inheritance
- [class="light"] Use augment to add properties to an existing object
%
